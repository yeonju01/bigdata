## DML
: 데이터 검색, 수정 등
INSERT : 데이터베이스 객체에 데이터 입력
DELETE : 데이터베이스 객체의 데이터 삭제
UPDATE : 데이터베이스 객체 안의 데이터 수정 
SELECT : 데이터베이스 객체로부터 데이터 검색

## DDL
: 데이터와 그 구조를 정의
CREATE : 데이터베이스 객체 생성
DROP : 데이터베이스 객체 삭제
ALTER : 데이터베이스 객체 재정의
TRUNCATE : 데이터베이스 객체 내용 삭제

## DCL
: 데이터베이스 사용자의 권한 제어
GRANT : 데이터베이스 객체에 권한 부여
REVOKE : 데이터베이스 객체의 권한 취소
COMMIT : 트랜잭션 확정
ROLLBACK : 트랜잭션 취소
SAVEPOINT : 복귀지점 설정

## 데이터형
- CHAR(size) ; 고정 길이 문자 데이터. VARCHAR2와 동일한 형태의 자료를 저장할 수 있고, 입력된 자료의 길이와는 상관없이 정해진 길이만큼 저장 영역 차지. 최소 크기는 1
- VARCHAR2(size) ; Up to 4000 Bytes 가변 길이 문자 데이터. 실제 입력된 문자열의 길이만큼 저장 영역을 차지. 최대 크기는 명시해야 하며, 최소 크기는 1
- NUMBER ; Internal Number Format 최고 40자리까지의 숫자를 저장할 수 있습니다. 이때 소수점이나 부호는 길이에 포함되지 않는다
- NUMBER(w) ; W자리까지의 수치로 최대 38자리까지 가능하다. (38자리가 유효 숫자이다.)
- NUMBER(w, d) ; W는 전체 길이, d는 소수점 이하 자릿수이다. 소수점은 자릿수에 포함되지 않는다.
- DATE : BC 4712년 1월 1일~AD 4712년 12월 31일까지의 날짜
- LOB : 2GB까지의 가변 길이 바이너리 데이터를 저장시킬 수 있습니다. 이미지 문서, 실행 파일을 저장할 수 있습니다

## 데이터 딕셔너리, 데이터 딕셔너리 뷰
:  사용자가 테이블을 생성하거나 사용자를 변경하는 등의 작업을 할 때 데이터베이스 서버에 의해 자동으로 갱신되는 테이블, 수정 및 삭제 불가
- DBA_XXXX : 데이터베이스 관리자만 접근 가능한 객체 등의 정보 조회 
- ALL_XXXX : 자신 계정 소유 또는 권한을 부여 받은 객체 등에 관한 정보 조회
- USER_XXXX : 자신의 계정이 소유한 객체 등에 관한 정보 조회
- ALL_ 데이터 딕셔너리 뷰 : 현재 계정이 접근 가능한 객체, 즉 자신 계정의 소유이거나 접근 권한을 부여 받은 타계정의 객체 등을 조회 할 수 있는 데이터 딕셔너리 뷰
